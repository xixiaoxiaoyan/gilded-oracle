# GLM-5.1 集成设置指南

## 概述

您的雷诺曼神谕应用现在已集成智谱AI的GLM-5.1模型，用于提供更智能、更个性化的占卜解读。

## 设置步骤

### 1. 获取API密钥

1. 访问智谱AI开放平台：https://open.bigmodel.cn/
2. 注册/登录账户
3. 进入"API密钥"页面：https://open.bigmodel.cn/usercenter/apikeys
4. 点击"创建API密钥"生成您的密钥

### 2. 配置API密钥

1. 打开项目根目录的 `.env.local` 文件
2. 将 `your_api_key_here` 替换为您的实际API密钥：

```env
ZHIPUAI_API_KEY=你的实际API密钥
```

### 3. 重启开发服务器

如果您正在运行开发服务器，请重启以加载新的环境变量：

```bash
# 停止当前服务器 (Ctrl+C)
# 重新启动
npm run dev
```

## 工作原理

- **主系统**：使用GLM-5.1 AI模型生成个性化的占卜解读
- **备用系统**：如果API调用失败，自动切换到传统规则系统
- **数据安全**：所有API调用都在服务器端进行，API密钥不会暴露给用户

## API调用示例

系统会向GLM-5.1发送以下信息：

- 用户的问题
- 抽出的牌（编号、名称、描述）
- 牌阵类型

GLM-5.1会返回完整的雷诺曼占卜解读，包括：
- 单张牌解读
- 牌的组合解读
- 综合分析
- 时间预测
- 最终结论和建议

## 费用说明

- GLM-5.1按使用量计费
- 建议查看智谱AI的定价页面：https://open.bigmodel.cn/pricing
- 每次占卜大约消耗数百tokens，成本较低

## 故障排除

### 问题：解读仍然使用传统规则系统

**可能原因**：
- API密钥未正确配置
- 网络连接问题
- API额度不足

**解决方法**：
1. 检查 `.env.local` 文件中的API密钥
2. 查看浏览器控制台的错误信息
3. 确认智谱AI账户有足够额度

### 问题：解读生成很慢

**可能原因**：
- GLM-5.1 API响应时间
- 网络延迟

**解决方法**：
- 这是正常现象，通常需要几秒钟
- 应用已设置加载状态显示"GLM-5.1 正在解读牌意..."
